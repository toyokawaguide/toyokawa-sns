# Google Sheets 移行手順書（社長作業 15分）

xlsx ローカル管理から Google Sheets 管理に切替えるための手順。
移行後は社長が xlsx 更新→push を手動でやる必要がなくなる（Sheets を編集するだけで自動で cron に反映）。

## 全体像

社長作業は**3つだけ**：
1. Google Sheet 作成＋CSV からコピペ（10分）
2. Apps Script デプロイ（3分）
3. GitHub Secrets 登録（2分）

私側（コト）は全部準備済み。コードもデプロイ済み。社長作業が終わった瞬間から Sheets モードが有効になります。

## STEP 1：Google Sheet を作成（10分）

### 1-1. 新規 Sheet 作成

https://sheets.new でブラウザで新しい Google Sheet を開く

→ 名前を「**豊川ガイド占い管理**」に設定（左上のタイトル「無題のスプレッドシート」をクリックして変更）

### 1-2. シートを15個作成

下のタブ（最初は「シート1」だけ）を右クリック→「**コピー**」or「+」ボタンで新規追加して、以下の名前のタブを作る：

#### ラッキースポット管理関連（7タブ）
- `ラッキースポット入力` ← 一番使う・社長が日々編集
- `マスタ`
- `使い方_必読`
- `候補リスト_リサーチ`
- `配信ログ`
- `過去記事抽出_2026-05-04`
- `ラッキースポット入力_old_2026-05-07`

#### 生まれ年グループ関連（4タブ）
- `生まれ年運用ルール_必読`
- `生まれ年グループ一覧`
- `生まれ年配信スケジュール` ← cron が読む
- `生まれ年全年リスト_時系列`

#### 町名グループ関連（4タブ）
- `町名運用ルール_必読`
- `町名グループ一覧`
- `町名配信スケジュール` ← cron が読む
- `町名全町リスト_五十音順`

### 1-3. CSV から貼り付け

**各 CSV のコピペ方法：**
1. CSV をメモ帳で開く
2. Ctrl+A で全選択 → Ctrl+C でコピー
3. Google Sheet の該当タブを開いて A1 セルをクリック
4. Ctrl+V で貼り付け
5. 「貼り付け→分割」ボタンが出たら「カンマ」で分割
6. 完了

⚠️ もし表示が崩れたら：「データ」→「テキストを列に分割」→ 区切り文字「カンマ」

## STEP 2：Apps Script を設定（3分）

### 2-1. Apps Script を開く

Google Sheet 画面で **「拡張機能」→「Apps Script」**

### 2-2. コードを貼り付け

開いた画面の `Code.gs` の中身を**全消去**して、以下ファイルの内容をコピペ：

📄 **`uranai/docs/apps_script_uranai.gs`** を開いて全内容コピー→Apps Script の Code.gs にペースト

### 2-3. SECRET_KEY を変更

コードの上の方にある：

```javascript
const SECRET_KEY = "CHANGE_ME_TO_RANDOM_STRING_xY9z2Q";
```

を、好きな**長い文字列**（英数字16文字以上）に変更。例えば：

```javascript
const SECRET_KEY = "uranai_2026_toyokawa_secret_xyz123";
```

この値は後で GitHub Secrets に登録するので、メモしておいてください。

### 2-4. 保存 → デプロイ

1. 上の **💾 保存ボタン** クリック
2. 右上の **「デプロイ」→「新しいデプロイ」**
3. ⚙ アイコン →「**ウェブアプリ**」を選択
4. 入力：
   - 説明：`占い配信API`
   - 次のユーザーとして実行：**自分**
   - アクセスできるユーザー：**全員**
5. 「**デプロイ**」クリック
6. 認証ダイアログ → 「アクセスを承認」（自分のGoogleアカウントを選んで「許可」）
7. デプロイ完了画面の「**ウェブアプリの URL**」をコピー

このURLは `https://script.google.com/macros/s/AKfycby.../exec` 形式。これも後で GitHub Secrets に登録。

## STEP 3：GitHub Secrets に登録（2分）

### 3-1. https://github.com/toyokawaguide/toyokawa-sns/settings/secrets/actions を開く

### 3-2. 「New repository secret」で2つ追加

| Name | Value |
|---|---|
| `URANAI_SHEETS_URL` | STEP 2-4 でコピーしたウェブアプリURL |
| `URANAI_SHEETS_SECRET` | STEP 2-3 で設定した SECRET_KEY と同じ値 |

---

## STEP 4：動作確認（私が実行）

社長が STEP 3 完了したら、「**Sheets移行完了**」と教えてください。私が以下を実行します：

1. GHA で手動 dry-run 実行（Sheets 経由でデータ取得できるか確認）
2. 既存の cron（毎朝6時）も Sheets 経由で動くか確認
3. xlsx ベースのフォールバックが効くか確認

問題なければ、xlsx を編集する代わりに Sheet を編集すれば、自動で cron に反映される運用に切替完了。

## トラブルシューティング

### Q. Apps Script の「ウェブアプリの URL」が表示されない
→ デプロイ画面で「アクセスできるユーザー：**全員**」を確認

### Q. GHA 実行で「sheet not found」エラー
→ Google Sheet のタブ名が手順書と完全一致しているか確認（半角全角・スペースに注意）

### Q. dry-run で xlsx 経由になってしまう
→ GitHub Secrets `URANAI_SHEETS_URL` が空 or 間違ってる可能性。再登録。
